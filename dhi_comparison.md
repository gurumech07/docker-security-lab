# Docker Hardened Image (DHI) Comparison Report

This report compares three Different Dockerfile approaches for the Python Flask application in this repository.

## Comparison Overview

| Feature                    | `Dockerfile.std`        | `Dockerfile` (Hardened) | `Dockerfile.dhi`               |
| :------------------------- | :---------------------- | :---------------------- | :----------------------------- |
| **Base Image**             | `python:3.9` (Official) | `python:3.11-slim`      | `dhi.io/python:3.12-alpine`    |
| **Image Size**             | Large (~900MB+)         | Small (~150MB)          | **Extra Small (~50MB)**        |
| **User**                   | `root` (Default)        | `appuser` (Custom)      | `appuser` (Built-in)           |
| **Vulnerabilities (CVEs)** | Many (Standard)         | Fewer (Slim)            | **Near-Zero (Hardened)**       |
| **Attack Surface**         | High (Full OS)          | Medium (Minimal)        | **Very Low (Distroless-like)** |
| **Multi-Stage Build**      | No                      | Yes                     | Yes                            |
| **Security Headers**       | Implemented in `app.py` | Implemented in `app.py` | Implemented in `app.py`        |

## Key Differences

### 1. `Dockerfile.std` (The "Naïve" Approach)

- **Pros:** Extremely simple.
- **Cons:** High CVE count, runs as root, large image size. Not suitable for production.

### 2. `Dockerfile` (Manual Hardening)

- **Pros:** Significant improvement. Uses multi-stage builds and a non-root user.
- **Cons:** Still contains common OS utilities that increase attack surface. Requires manual maintenance of security patches for the base image.

### 3. `Dockerfile.dhi` (The Modern Enterprise Standard)

- **Pros:**
  - **Minimalism:** Stripped of unnecessary shells and binaries.
  - **Proven Security:** Provided by Docker with SBOM and VEX signatures.
  - **Compliance-Ready:** SLSA Level 3 compliant base.
- **Cons:** Requires registry authentication (`dhi.io`).

## Conclusion

Transitioning from a standard `Dockerfile` to a manually hardened one is the first step. However, using **Docker Hardened Images (DHI)** provides the highest level of security with minimal maintenance effort by shifting the burden of base image security to Docker’s curated ecosystem.
